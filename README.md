## About

I'm a software engineering student at **Bontang Institute of Technology**, in East Kalimantan, Indonesia — building full-stack products that solve problems I can actually point to, not tutorial clones.

It started small: a static site for an Earth Day poster competition run by my own campus. Then it became something closer to real work — a dashboard a tutoring business actually uses to manage schedules and payments. Then it became something I'm still building: a full-stack EdTech platform, in Bahasa Indonesia, with its own auth system, its own database schema, and an AI assistant wired into the editor — because most JavaScript courses are written for an audience that isn't mine.

I believe in shipping production habits early. Token rotation, input validation, and rate limiting don't get to wait until a project is "serious enough" — they go in from commit one, even on a personal learning repo. That's the part of the brand promise — *Code · Learn · Improve* — that I take literally: the improving never stops, including on code nobody asked me to harden.

Right now I'm heading toward deeper backend architecture and AI-assisted developer tooling, while keeping every project grounded in a real user — a student, a small business, a community — rather than a demo that only has to impress a portfolio reviewer.

## How I Build

Each principle below is paired with where it actually shows up in my repos — not a slogan on its own.

| Principle | Where it's proven |
|---|---|
| **Security by default, not by deadline** | JWT access + rotating httpOnly refresh tokens, bcrypt(12), Zod validation on every endpoint, Helmet headers — in `JSCraft`, a *learning* project. |
| **Architecture before features** | `JSCraft` is a Turborepo monorepo: `apps/web`, `apps/api`, `packages/types`, `packages/content` — separated before the feature count justified it. |
| **Build for one real user, not a demo** | `SNA-studyhub` exists because a tutoring business needed schedules, payments, and analytics in one dashboard — not because a portfolio needed a project. |
| **Lower the barrier, don't just clear it for myself** | `JSCraft`'s entire curriculum, UI, and docs are written in Bahasa Indonesia — built for developers the English-first EdTech world skips. |
| **Ship the roadmap in the open** | Every repo README carries a visible, checkbox-tracked status table — including the parts that are still planned, not just the finished parts. |

JSCraft is the clearest bet on that idea so far: a full curriculum, in Bahasa Indonesia, with an AI assistant that explains rather than just autocompletes.
